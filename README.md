## Project Overview

This project is a structured audit of **Git**, the world's most widely used distributed version control system. It explores Git from technical, philosophical, and practical perspectives as part of an Open Source Software course.

The project highlights:

* Origin and development of Git
* Open-source philosophy and licensing (GPL v2)
* Linux system integration
* Git ecosystem and dependencies
* Comparison with proprietary tools
* Practical implementation using shell scripts

##  Project Structure

###  Part A – Origin and Philosophy

* History of Git (created by Linus Torvalds in 2005)
* Problem Git solved in Linux kernel development
* GPL v2 License explanation
* Ethics of open-source software

###  Part B – Linux Footprint

* Installation using package manager
* Important directories and configuration files
* System-level integration
* Update and security model

### 🔹 Part C – FOSS Ecosystem

* Dependencies (zlib, libcurl, OpenSSL, etc.)
* Tools built on Git (GitHub, GitLab, CI/CD tools)
* Role in modern DevOps

### 🔹 Part D – Critical Analysis

* Open Source vs Proprietary comparison
* Advantages of Git
* Real-world relevance and recommendation

##  Shell Scripts Included

### 3️⃣ Git Repository Analyzer

* Active branch
* Contributors

### 4️⃣ Log File Scanner

Searches log files for specific keywords.

##  Comparison (Open Source vs Proprietary)

| Feature        | Git  | Proprietary Tools |
| -------------- | ---- | ----------------- |
| Cost           | Free | Paid              |
| Source Code    | Open | Closed            |
| Flexibility    | High | Limited           |
